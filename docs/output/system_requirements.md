## システム概要

RUNdio は、ソロランナーの**マンネリ緩和**と**モチベ維持**を目的とし、走行セッションの**目標時間または距離**に合わせた**一続きのラジオ風音声体験**を、**走り始める前**に用意するモバイル最適化 Web である。本フェーズのゴールは**社内プレゼン**向けに、**約6名**が個別アカウントで触れ、**PC（LP 内スマホ枠）**と**iOS Safari**から同等体験を示すこと。収益化は主目的としないが、**プロダクト品質に見える UI・音声・安定性**を求める。

1. **システムの目的**
   - **課題**: 汎用音楽・ラジオでは得にくい「走りの文脈に合った音声」「次に何が聞けるか」という期待。
   - **成果**: オンボーディング〜セッション作成〜（生成待ち）〜再生までの**一連の UX**を実機で提示。
   - **ビジネスインパクト**: 社内における**課題提起・技術＋UX の説得力**（プレゼン主軸は **UX**）。

2. **主な機能**
   - **コア**: Clerk 認証、ランナープロフィール（嗜好・目的・コースイメージ）、目標**時間または距離**によるセッション作成、放送ジョブ（将来は LLM+TTS、現状はデモ用**固定音声**含む）、再生 UI。
   - **差別化**: パーソナル設定に基づくトーン設計（コンテンツ層）、**LP 内デバイス枠**によるプレゼン体験。
   - **付加**: 生成待ちの**進行演出**（デモ用遷移）、距離→時間の**決め打ち想定ペース**（6:00/km）換算。

3. **提供する価値**
   - **ユーザー**: 自分向けの「今日の放送」という**選択肢**、自然な音声品質（高品質 TTS 前提、デモはアップロード音声可）。
   - **ビジネス（社内）**: マルチユーザー・サーバー側処理の**説明可能性**（補助線）。
   - **社会**: 本フェーズでは限定的（健康配慮の**一般表現**に留める）。

4. **実現方法の概要**
   - **Next.js（App Router）** + **TypeScript** + **Tailwind CSS v4**、**Clerk** 認証、**Supabase（PostgreSQL）**、将来 **Storage**・**外部 LLM/TTS**。**Vercel** 想定ホスティング。
   - クライアントは **iPhone 幅想定**、LP は **iframe / 同一オリジン**でスマホ枠表示。

## 機能要件

- **認証**: サインアップ／サインイン／サインアウト（Clerk）。Supabase `users` との同期。
- **オンボーディング**: `runner_profiles` への嗜好・目的・コースの保存（必須度は運用で調整可）。
- **セッション作成**: 時間目標または距離目標（排他選択）。距離は **6:00/km** 定数で所要時間・ジョブの `target_duration_min` を算定。
- **ジョブ**: `broadcast_jobs` の作成・参照。ステータス（pending/processing/ready/failed）。MVP スタブでは **ready + 固定パス** でよい。
- **待機 UI**: 生成進捗のプレースホルダー（デモでは短い遷移演出）。
- **再生**: `<audio>` による再生、iOS の自動再生制約を考慮（ユーザー操作起点）。
- **LP**: コンセプト、**StartCta**（未ログイン→サインイン、ログイン済→/home）、スマホ枠デモ。

## 非機能要件

| 区分 | 要件 | 根拠・仮定 |
|------|------|------------|
| パフォーマンス | 主要画面 **3 秒以内**の体感、LP は LCP 重視 | 6 ユーザー規模 |
| セキュリティ | **API キーはサーバーのみ**、HTTPS、Clerk セッション | 講義・デモでも最低限 |
| 可用性 | デモ中の落ちを避ける。**固定音声フォールバック** | プレゼンリスク |
| スケーラビリティ | 水平スケールは**後続**（Vercel/Supabase の標準機能に依存） | MVP |
| 品質 | **下ネタ禁止**、励ましは**一般表現**。ブランド名は当面許容 | product-requirements |
| A11y | セマンティック HTML、フォームラベル、音声コントロール | 講義推奨 |

## 技術スタック

| 層 | 採用 | 理由 |
|----|------|------|
| フロント | **Next.js 15+**（App Router）、React 19 | 講義指定に近い。RSC/Server Actions でデータ取得と認証の統合 |
| 言語 | TypeScript | 型安全・チーム開発 |
| スタイル | Tailwind CSS v4 | 既存コードベース整合 |
| 認証 | **Clerk** | 既存マイグレーション・middleware 整合 |
| DB | **Supabase（PostgreSQL）** | `users`, `runner_profiles`, `broadcast_jobs` |
| ホスティング | **Vercel**（想定） | Next との親和 |
| 将来 | Supabase Storage、OpenAI 等 LLM、ElevenLabs 等 TTS | 長尺生成・コスト設計は別フェーズ |

※ 講義プロンプトの「Next.js 16」表記があるが、現リポジトリは **Next 15** 系で整合。アップグレードは別タスク。

## リスク分析

| カテゴリ | リスク | 対策 |

## UI/UXの方向性

- **デザインコンセプト**: ダーク基調・**シアンアクセント**、ミニマルだが安っぽく見えない余白。
- **トーン＆マナー**: 寄り添い・前向き、**ランナーに語りかける**口調（音声設計と一致）。
- **ターゲットデバイス**: **モバイルファースト（iOS Safari）**、LP は **デスクトップ＋スマホ枠**。
- **参考**: Apple Fitness+（情報密度）、Spotify（ダーク UI）、Nike Run Club（ラン文脈）※厳密なコピーはしない。
