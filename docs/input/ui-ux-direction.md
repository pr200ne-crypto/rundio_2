# UI/UX 方針 — RUNdio

## 1. 全体の方向性

| 項目 | 方針 |
|------|------|
| デザインテーマ | **ダーク基調**、**シアン（cyan）**をアクセント。夜間・屋外のラン前後でも目に優しい。 |
| キーワード | 落ち着き、信頼、**パーソナル**、ラジオ的な温かみ（UIはミニマル、**音声**に主役を譲る）。 |
| 参考イメージ | 高品質フィットネス／メディアアプリ（過剰な装飾は避ける）。 |

LP とアプリ内で**タイポグラフィ・色・角丸のルール**を共通化し、「別アプリ」と感じさせない。

---

## 2. ビジュアル

### カラー（目安）

- 背景: 深いグレー〜ネイビー（例: zinc-950 系）。
- 本文: zinc-50〜100。
- 補助テキスト: zinc-400〜500。
- アクセント: cyan-400〜500（CTA・進行状態）。

### タイポグラフィ

- システムフォントスタック + **Noto Sans JP**（読みやすさ優先）。
- 見出しはややタイト、本文は**行間ゆったり**（走る前にサッと読める）。

### アイコン・イラスト

- MVP は**線画アイコン**またはテキスト中心でも可。写真イラストは統一感が取れる場合のみ。

---

## 3. UX 原則

### 重視する体験

- **ワンタップの幅**: 親指で届くエリア、ボタン高さは最低 44px 相当。
- **待ちの透明性**: 生成中は**何が起きているか**（処理中／残り目安）を短い文言で。
- **失敗の扱い**: 技術用語より「もう一度試す」「時間を短くする」など**次の行動**が分かるコピー。
- **音声ファースト**: 画面情報は最小限。再生画面は**大きな再生コントロール**。

### 避けたい UX

- 機械っぽいコピー（「エラーコード 500」の丸出し等）の多用。
- 生成完了前に**走れ**と促すフロー（**事前生成**がコンセプトの核）。

---

## 4. 画面構成（主要）

| 画面 | UX メモ |
|------|---------|
| LP | 一文で価値、**スマホ枠**が視覚的フォーカル。 |
| オンボーディング | 項目は少なく、**任意スキップは将来**（MVP は必須でも可）。 |
| セッション作成 | **時間 OR 距離**が一目で分かるトグルまたはタブ。 |
| 待機 | 進捗、**推奨**: 軽いアニメまたはステップ表示。 |
| 再生 | 再生／一時停止、**経過時間**、（任意）残り時間。 |

画面遷移の詳細は [user-flows.md](./user-flows.md)。

---

## 5. インタラクション

| 項目 | 方針 |
|------|------|
| アニメーション | **控えめ**（200〜300ms、opacity / translate）。 |
| フィードバック | ボタンは `active` スケールまたは背景変化。 |
| エラー表示 | インライン + 下部トーストでも可。 |

---

## 6. 音声体験（UX の一部）

- **品質**: 高品質 TTS（例: ElevenLabs）。**Mei - Friendly, Clear and Soft** 系の**明瞭で柔らかい**方向性を参考。
- **台本**: 走っている人に向けた**二人称・現在形**。長すぎる一文を避け、**息継ぎの間**を意識。
- **NG**: ガチな機械朗読、不自然なイントネーションだらけの長文。
- **コンテンツ制約（合意）**: **下ネタ禁止**。**ブランド名は当面OK**。心拍等のデータがない前提のため、励ましは**当たり障りのない一般表現**（例:「いいペースですね」「無理せず」）に寄せる。
- **言語**: **日本語のみ**。
- **デモ**: **アップロード済み音声**を流す前提でも、UI・コピーは上記に整合させる。

---

## 7. LP 内スマホ枠

- 幅 **360〜390px**、角丸フレームで**実機の没入感**。
- iframe 内も**同じトークン**（色・フォント）を共有。
